# 실무에서 자주 사용하는 HTML 필수 태그
적재적소.. 태그를 잘 사용해 작성해야 웹 페이지 구성을 잘 할 수 있다.  
150여 개의 태그들이 존재하며 (https://www.w3.org 에서 확인 가능), 주로 많이 사용하는 태그들을 정리.

## 텍스트 작성 시 사용하는 태그들
### `<hn>` 태그
```html
<h1>Heading level 1</h1>
<h2>Heading level 2</h2>
<h3>Heading level 3</h3>
<h4>Heading level 4</h4>
<h5>Heading level 5</h5>
<h6>Heading level 6</h6>
```
'Heading'의 약자 h, n은 숫자 중요도를 나타낸다. h1 > h6
> 글씨 크기 차이도 난다.  

`<hn>` 태그로 작성된 텍스트는 검색 엔진에서 키워드로 인식되기에 본문에서 핵심이 되는 내용으로 제목 및 주제를 잘 선택해 작성  
또, h1 태그부터 순서대로 사용하는 게 좋다.  
(검색 엔진에서 단계적으로 검색하기 때문에 중간에 건너 띄워 사용할 경우 건너뛴 숫자부터 태그들을 검색하지 않는다.)
### `<p>` 태그
```html
<p>내용</p>
```
제목, 주제를 나타내는 텍스트가 아닌 본문에서 자주 사용한다.  
문단 기준으로 나눠지며 블록요소이기에 <p> 태그 기준으로 줄 바꿈이 이루어진다.  
태그 없이 그냥 작성할 시, 다닥다닥 붙어서 출력 된다.

### `<br>` 태그
```html
<br>
```
HTML에서 줄 바꿈 할 때 사용하는 태그.

### `<blockquote>` 태그
```html
<blockquote>문단 단위 인용문 작성</blockquote>

<!--출처가 확실한 인용문은 cite 속성으로 출처 경로 명시-->
<blockquote cite="출처 URL"> 문단 단위 인용문</blockquote>
```
`cite` 속성으로 출처 링크를 명시할 수 있다.  
실제 웹 브라우저에 표기는 안되지만 브라우저, 검색엔진 등에 알려주기 위함이라고 한다.  
(gpt에 물어봄)  
이 태그를 사용할 때 `<p>` 태그를 포함해서 사용해야 하며, `<p>` 태그에 `<blockquote>` 태그를 들여 쓸 수 없다.  
```html
<blockquote cite="https://haze.mingizeok.com">
    <p>
        상상을 구현하고자 웹개발 공부를 했습니다. <br>
        Python 기반의 Django, FastAPI 위주로 작업합니다.<br>
        뭐가 될진 모르겠지만, 뭐라도 되어보겠습니다. ㅎㅅㅎ
    </p>
</blockquote>
```
> 약간 모서리에 공백이 잡히는 듯 하다.
### `<q>` 태그
```html
<q>짧은 인용문</q>

<q cite="출처 URL">짧은 인용문</q>
```
문단 안에 텍스트 단위의 짧은 인용문 작성 때 사용.  
`<blockquote>` 태그처럼 `cite` 속성을 사용할 수 있다.  
> `<p>` 안에서도 사용할 수 있다.
```html
<p>
    밍기적 실험실은 <q ctie="https://haze.mingizeok.com">상상을 구현하고자 웹 개발 공부</q>를 시작한 사람이 진행한 프로젝트들이 모여있는 공간이다.
</p>
```
❗️`<q>`태그를 사용한 텍스트는 자동으로 "큰따옴표"로 묶인다.
### `<ins>` `<del>` 태그
```html
<ins>추가 텍스트</ins>
<del>삭제 텍스트</del>

<p>
    뭘 써야할지 잘 모르겠네.. <del>shit</del>아이디어 부족...<ins>아무말 적어봄</ins>
</p>
```
`<ins>` 태그는 새로이 추가된 텍스트 (밑줄로 강조됨), `<del>` 태그는 기존에 있던 텍스트를 삭제(취소선)할 때 사용한다.
### `<sub>` `<sup>` 태그
```html
<sub>아래 첨자</sub>
<sup>위 첨자</sup>

<p>
    공기의 원소 기호는 H<sub>2</sub>0<br>
    4<sup>2</sup>은 16입니다.
</p>
```
각각 아래, 위 첨자에 해당하는 텍스트 작성할 때 사용한다.  
> 숫자말고도 가능하다..!!

## 텍스트 강조하기
택스트 강조하는 `<b>` `<i>` 태그는 각각 `<strong>`, `<em>` 태그와 비슷, 잘 사용되지 않고 있다.  
-> 주로 `<strong>` `<em>` 사용한다.  
### `<strong>` 태그
```html
<strong>택스트 강조</strong>
```
볼드체, 택스트를 굵게 표시 및 웹 브라우저에 중요한 부분이라고 알려주는 태그.  
중복해서 사용할 수 있다. (중첩될수록 구조적으로 의미를 더 강조하게 된다. 시각적으로는 차이 없음)  
```html
<strong>✍🏼링크 추가</strong><br> <!--강조하고 싶은 부분-->
<a href="coding/coding.html">
	CODING
</a>
<br>
<a href="https://haze.mingizeok.com" target="_blank" title="실험실">
	안개를 걷어내고 실험실 찾는 중
</a>

<!--부분 강조-->
<p><strong>안개</strong>를 걷어내고 실험실 찾는 중</p>
<em> 태그
<em>기울어진 텍스트</em>
<em> 태그로는 기울기로 텍스트를 강조해 줄 수 있다.
<strong> 태그처럼 중첩해서 사용 가능하다.
```
`<strong>` `<em>`태그 사용. 여기서는 `<strong>`이 조금 미미하게 보인다.  

## 그룹 짓기 **(공간 분할 태그)**
레이아웃 구성 시, 관련 요소들끼리 그룹으로 묶어 작성하면 편하다.  
다른 영역과 분리한다는 의미로 '공간 분할'이라고 한다.   
`<div>` 태그와 `<span>`태그가 있다.  
### `<div>` 태그
여러 요소들을 묶을 때 사용한다. (블록, 인라인 요소 모두 가능)  
```html
<div id="profile">
	<h2>::PROFILE::</h2>
	NAME : SHINEDAY <br>
	LEVEL : 2&#9746; <br>
	BIRTH : WINTER <br>
	MBTI : INFP <br>
	CLASS : <s>INSTRUCTOR</s> HOME PROTECTOR <br>
	SKILL : ?
</div>
```
(전에 만들었던 프로필 페이지 코드를 가져왔다.)  
`<div>` 태그를 사용해 여러 요소들을 그룹으로 묶어 CSS 스타일 적용할 때 사용할 수 있다.  
(적용 단위로 그룹 묶는 것도 좋지 않을까 하는 조심스러운 생각)  
### `<span>` 태그
인라인 요소에서 사용되는 태그.  
```html
CLASS : <span>INSTRUCTOR</span> HOME PROTECTOR <br>

MP : <span id="result"></span> <br>
```
부분적으로 포인트를 줄 때 이 태그를 사용해서 스타일 적용을 시킬 수 있다.  
아래의 경우는 따로 자바스크립트로 숫자 넣는 기능 만들어서 불러오는 방식으로 사용된 듯하다.
(gpt의 도움을 많이 받았다.)  

## 목록 만들기
목차 및 메뉴 등 다양한 곳에서 목록을 만들 때 사용하는 태그들.  
### `<ul>` `<ol>` `<li>` 태그 : 순서형 비순서형  
```html
<!--비순서형 목록-->
    <li>목록 내용1</li>
    <li>목록 내용2</li>
    <li>목록 내용3</li>
</ul>

<!--순서형 목록-->
<ol>
    <li>목록 내용1</li>
    <li>목록 내용2</li>
    <li>목록 내용3</li>
</ol>
```
`<ul>`, `<ol>`태그 안에 `<li>`태그를 사용해 목록 리스트를 만들 수 있다.  
`<ul>` 태그는 비순서형, `<ol>` 태그는 순서형(숫자로 매겨진다.) 리스트를 뜻하는 태그다.  
항목들은 `<li>` 태그를 사용해 작성한다.  

> 웹 브라우저를 통해 확인하면 차이를 확실히 알 수 있다.

### `<dl>` `<dt>` `<dd>` 태그 : 정의형 목록
```html
<dl>
    <dt>용어</dt>
    <dd>용어 설명<dd>
</dl>
```
인용 태그들이랑 비슷한 느낌으로 사용이 되는 거 같기도 하다. (웹 브라우저, 검색엔진에 알려주는 목적..?)  
`<dt>`, `<dd>` 용어와 용어 설명 태그를 `<dl>`로 묶어 정의한다.  
```html
<dl>
    <dt>밍기적 실험실</dt>
    <dd>밍기적에서 벗어나 뭐라도 좀 해보라는 의미로 네이밍. 다양한 웹 개발 프로젝트 및 기타 등등에서 사용하고 있다.</dd>
    <dt>주문</dt>
    <dd>밍기적 실험실에서 운영중인 랜덤메뉴 추천 서비스</dd>
    <dt>밍기적 숫자야구</dt>
    <dd>밍기적 실험실에서 운영중인 숫자야구 게임</dd>
</dl>
```
## 링크와 이미지 넣기
### `<a>` 태그 : 링크
HTML 내부나 외부 링크를 생성, 연결을 한다.
```html
<a href="대상경로" target="연결방식" title="링크 설명"</a>
```
`<a>` 태그를 사용할 때 'href'속성은 필수로 사용해 경로를 지정, 'target', 'title' 속성은 선택 사항이다.  
'href'는 URL, 내부문서의 id 값일 수 있다.  
```html
<a href="coding/coding.html">CODING</a>

<a href="https://haze.mingizeok.com" target="_blank" title="실험실">안개를 걷어내고 실험실 찾는 중</a>
```
'target'은 연결방식을 지정하는 데, 주로 `_blank` 속성 값을 사용한다. (링크 클릭 시 새로운 탭으로 열린다.)
### `<img>` 태그 : 이미지
```html
<img src="이미지 경로" alt="이미지 설명">
```
`<a>` 태그에서 'href' 속성과 비슷하게 'src' (source) 속성으로 이미지 경로를 지정해 줄 수 있다.  
이미지 경로는 웹 브라우저에서 실행되는 HTML 파일 위치가 기준이다.  
```html
<img src="photo.jpeg" alt="부산 바닷가에서 일출 배경으로 찍은 사진">
```
'alt'속성은 이미지 설명을 기입한다. 이미지가 로딩이 안될 경우 이 속성에 작성한 텍스트가 출력이 된다.  
'src'속성과 'alt'속성은 웹 표준에서 필수 사용으로 권고하고 있다.  
같은 폴더 내에 이미지가 있을 경우 이름만 적어도 상관없지만 다른 위치에 있는 경우 약속된 기호로 이미지의 위치를 나타낼 수 있다.  
```html
<!--같은 위치-->
<img src="이미지명">

<!--상위 폴더에 위치-->
<img src="../이미지명">

<!--하위 폴더에 위치-->
<img src="하위폴더이름/이미지명">
```
`.` 점 하나는 현 위치를 뜻하고 `..` 점 두 개는 상위 폴더를 뜻한다.  
현재 폴더를 뜻할 때 별도의 표기 없이 이미지 이름으로만 표기를 해도 인식이 된다.  
상위폴더를 뜻하는 기호는 중복으로도 사용이 가능하다. `../../이미지명`  

> 이미지를 불러오지 못할때 입력한 텍스트가 출력. 따로 박스로 메세지 뜨기도 했던거 같다.

### 이미지 링크 : `<a>` `<img>` 태그 활용
```html
<a href="storageBox/storageBox.html">
	<img src="photo.jpeg" alt="부산 바닷가에서 일출 배경으로 찍은 사진">
</a>
```
`<img>` 태그를 `<a>` 태그로 감싸 '이미지 링크'. 즉, 이미지 클릭 시 해당 특정 링크로 이동할 수 있게끔 연결시켜줄 수 있다.

## 멀티미디어 설정하기
### `<audio>` 태그
```html
<audio src="오디오 파일 경로" controls></audio>
```
'src' 속성으로 함께 사용한다.  
오디오 컨트롤 패널을 웹 브라우저에서 볼 수 있도록 controls 속성을 설정한다.  
웹 브라우저 마다 지원하는 포맷이 다르다. (mp3지원 브라우저가 많다)  
### `<video>` 태그
```html
<video src="비디오 파일 경로" controls></video>
```
`<audio>` 태그처럼 'src'속성과 함께 사용되며 컨트롤 패널 속성도 추가해야 한다.  
비디오도 지원하는 포맷이 브라우저마다 차이가 있는데, 많은 웹 브라우저에서 mp4 포맷을 지원한다.  
### `<source>` 태그
```html
<audio controls>
	<source src="파일 경로" type="미디어 타입">
</audio>
<video controls>
	<source src="파일 경로" type="미디어 타입">
</video>
```
`<source>` 태그로 경로 지정과 미디어타입을 지정할 수 있다.  
웹 브라우저마다 지원하는 포맷들이 다르다보니 우선 순위를 지정해 먼저 재생을 시도해보고 안되면 후순위로 재생을 시도하는 방법으로 활용할 수 있다.  
```html
<audio controls>
    <source src="경로" type="audio/wav"> <!-- wav 먼저 시도 -->
    <source src="경로" type="audio/mp3"> <!-- mp3 시도 -->
    지원하지 않는 웹 브라우저입니다. <!-- 둘다 안될 경우 이 텍스트 노출 -->
</audio>
```
위에서부터 명시된 포맷으로 시도, 안될 경우 밑에 작성한 텍스트가 출력된다.  
### 글로벌 속성 global attribute
태그 종류 상관 없이 모든 태그에서 공통으로 사용할 수 있는 속성들이 있다.  
#### class
```html
<p class="클래스명"></p>
```
CSS에서 클래스 선택자로 사용.  
여러 요소가 중복해 가져갈 수 있다.  
#### id
```html
<p id="아이디명"></p>
```
CSS에서 아이디 선택자로 사용.  
class와 마찬가지로 여러 요소가 중복해 가져갈 수 있다.  
#### style
인라인 스타일 지정. CSS 코드를 인라인으로 작성할 때 사용한다.  
#### title
```html
<p title="추가설명"></p>
```
추가 정보 지정.  
마우스를 요소 위에 올리면 툴팁 tooltip으로 추가 정보가 표시.  
#### lang
```html
<html lang="ko">
```
요소에 사용한 텍스트 언어 정보 지정.  
기본 언어 코드 설정으로 웹 접근성을 높일 수 있다.  
#### hidden
요소를 화면에서 감춘다.  
#### data-*
```html
<p data-이름="값"></p>
```
사용자가 임의의 속성을 만들 수 있다. (커스텀 속성)
