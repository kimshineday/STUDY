# Javascript
웹 브라우저가 HTML과 CSS를 가져와 해석된 내용을 표시한다.  
### 정적 웹 Static Web
HTML과 CSS가 직접 수정이 되기 전까지는 기존 작성된 내용이 그대로 유지가 된다.  
웹 브라우저에서 한 번 해석되어 화면에 표현이 되고나면 수정을 할 수 없다.  
다시 코드 반영 후, 재실행해야한다.  

### 동적 웹 Dynamic Web
웹 브라우저에 표시된 내용을 인위적으로 조작 및 상호작용할 수 있다.  
특정 어떤 행동을 할 때 웹 브라우저에서 반응을 줄 수 있다.  
이를 JavaScript를 사용해서 구현할 수 있다.  

## 코드 작성 방법
### 내부 스크립트 Internal Script
```html
<body>
    <script>
    	// 코드 작성
    </script>
</body>
```
HTML 에서 `<script>` 태그를 사용해 내부에서 자바스크립트 코드를 작성하는 방법이다.  
### 외부 스크립트 External Script
```html
<body>
	<script src="js파일경로"></script>
</body>
```
따로 스크립트 파일을 만들어 연결시키는 방법도 있다.  
이때 src 속성을 사용한다.  
#### script 태그 사용 위치
```html
<body>
	<!-- body 코드들 -->
    
    <!-- 외부 스크립트 -->
    <script src="경로"></script>

    <script>
        <!-- 내부 스크립트 -->
    </script>
</body>
```
HTML CSS 해석이 끝나는 시점인 `<body>`의 끝자락에 위치한다.  

## 코드 실행
크게 VSC에서 확장프로그램 Code Runner를 설치해서 사용하는 방법과 콘솔 창에서 실행하는 방법이 있다.  
### 콘솔창에서 실행
```js
document.write('콘솔창에서 실행하기') // 웹 브라우저에서 텍스트 표시
```
개발자도구 developer tools > console 에서 실행할 수 있다.  
비어있던 화면에 코드 실행 후 화면에 출력이 되는걸 볼 수 있다.  

### VSC 확장 프로그램 사용
js 파일에서  ▶️ 버튼을 눌러 코드를 실행할 수 있다.
```js
console.log('확장 프로그램 사용') // 에디터에서 바로 실행
```
output창에서 출력 값을 확인할 수 있다.  

## 주석
```js
// 주석 내용

/* 
주석 내용을 여러줄 작성할 때 사용.
CSS 주석 처리와 동일하다.
*/
```
두가지 방법으로 주석을 처리할 수 있다.  

## 오류 확인 방법
```js
// 오류 났을 경우 확인. 오타로 에러 발생
consol.log('오류 억지로 만들기')
```
자바스크립트는 인터프리터(interpreter) 방식으로 코드를 한 번에 한 줄씩 실행하게 된다.  
에러 발생 시 실행을 멈추고, 발생 시점과 위치를 알려준다.  
클릭하면 어디서 에러났는지 위치 확인 할 수 있다.
